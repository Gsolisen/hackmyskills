```python
@dataclass
class SessionResult:
    total: int = 0
    correct: int = 0  # Good(3) or Easy(4)
    topic_stats: dict[str, dict] = field(default_factory=dict)
    ratings: list[int] = field(default_factory=list)

    def record(self, topic: str, rating_value: int) -> None:
        self.total += 1
        is_correct = rating_value >= 3
        if is_correct:
            self.correct += 1
        self.ratings.append(rating_value)
        ts = self.topic_stats.setdefault(topic, {"total": 0, "correct": 0})
        ts["total"] += 1
        if is_correct:
            ts["correct"] += 1

    @property
    def accuracy_pct(self) -> int:
        return round(100 * self.correct / self.total) if self.total else 0

    @property
    def xp(self) -> int:
        return self.total * 15  # Phase 2 placeholder; Phase 3 replaces with tier-weighted formula
```

3. `build_queue(daily_cap: int, topic: Optional[str] = None) -> list[Card]`:
   - `now = datetime.now(timezone.utc).replace(tzinfo=None)` — naive UTC, matches DB storage
   - If `topic` is not None: check `Card.select().where(Card.topic == topic).count()` first. If zero, raise a `ValueError(f"No cards found for topic '{topic}'")` so the caller can show the error panel.
   - Due cards: `Card.due.is_null(False) & (Card.due <= now)` ordered by `Card.due.asc()`
   - New cards: `Card.due.is_null(True)` ordered by `Card.topic.asc(), Card.id.asc()`
   - Fill up to `daily_cap`; return `queue[:daily_cap]`

4. `persist_rating(db_card: Card, rating: Rating) -> None`:
   - Load `FsrsCard.from_json(db_card.fsrs_state)` if fsrs_state else `FsrsCard()`
   - Call `review_card(fsrs_card, rating)` → `(updated_fsrs, review_log)`
   - Strip tzinfo: `new_due = updated_fsrs.due.replace(tzinfo=None)`
   - Update db_card fields: `fsrs_state`, `due`, `stability`, `difficulty`, `state`, `reps += 1`, `lapses += 1` if Again
   - Use `db.atomic()` context to save Card and create ReviewHistory atomically (prevents partial writes on Ctrl-C)

5. `compute_streak() -> int`:
   - Query ReviewHistory ordered by `reviewed_at.desc()`
   - Collect unique `date` values from `reviewed_at`
   - Walk backwards from `date.today()` using `timedelta(days=1)` (not manual day arithmetic)
   - Return count of consecutive days

6. `_render_progress(console: Console, card_num: int, total: int) -> None`:
   - Call `console.clear()`
   - Print dim line: `f"Card {card_num}/{total} {'─' * int(40 * card_num / total)} {int(100 * card_num / total)}%"` with `style="dim"`

7. `_wait_for_key(valid_keys: set[str] | None = None, _readkey=None) -> str`:
   - Default `_readkey` to `readchar.readkey` if None (lazy import: `import readchar; _readkey = readchar.readkey`)
   - Loop calling `_readkey()` until result is in `valid_keys` (or any key if `valid_keys` is None)
   - `readchar.readkey()` raises `KeyboardInterrupt` on Ctrl-C — let it propagate naturally

8. `run_session(topic: Optional[str] = None, _readkey=None) -> None`:
   - Call `load_config()` → `daily_cap = config["daily_cap"]`
   - Try `build_queue(daily_cap, topic)`. On `ValueError` (no cards for topic): print red Panel with error message and return.
   - If queue is empty: print "Nothing to review right now — come back tomorrow!" and return.
   - Wrap full session loop in `try/except KeyboardInterrupt` for clean quit.
   - Session loop: iterate cards with index, call `_render_progress()`, dispatch to handler by `card.question_type`, record result via `session.record()`, persist rating.
   - Handlers for each type are NOT implemented yet — use `raise NotImplementedError(card.question_type)` as placeholder. Wave 2 plans fill these in.
   - After loop (or KeyboardInterrupt): if `session.total >= 1`, call `_show_summary(session, is_partial=...)`. Always commit any already-reviewed cards (persist_rating handles DB writes per card, so partial sessions are already safe).
   - Handler dispatch: call `_handle_flashcard(card, session, _readkey)`, `_handle_command_fill(card, session, _readkey)`, `_handle_scenario(card, session, _readkey)`, `_handle_explain_concept(card, session, _readkey)` — define all four as stubs that `raise NotImplementedError`.
   - `_show_summary` stub: accepts `(session: SessionResult, is_partial: bool = False)` — `raise NotImplementedError` for now.

Imports needed: `from __future__ import annotations`, `dataclasses`, `datetime`, `typing`, `fsrs`, `rich.console`, `rich.panel`, `hms.models`, `hms.scheduler`, `hms.config`, `hms.db`.
